Sensor calibration

Different data serving modes may require different calibration values. Low data frequency may cause the esp8266 chip to get warm, it will warm up the air near the sensor. I.e. in a WEB SERVER or a Display ONLY mode the measured temperature might be higher than in the default data serving mode with a esp8266 chip deep sleep. If you use the default, HTTP or MQTT data sending configuration, you don't need to change here anything.
